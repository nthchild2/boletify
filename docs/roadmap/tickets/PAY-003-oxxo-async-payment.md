**Epic:** Payments

## Description

Implement OXXO cash payment flow: generate an OXXO payment reference (barcode + reference number) with a 72-hour expiry window, show the reference to the buyer, send the reference via email, and release held inventory when the payment window expires without payment.

This is the highest-priority payment integration — it unlocks the ~40% of buyers who don't have credit cards.

Phase 2 refs: B6 (OXXO payment), Phase 4 § 8.2 (OXXO payment flow), § 8.4 (OXXO convenience fee).

---

## Acceptance Criteria

- [ ] **AC1:** When buyer selects "OXXO" at checkout, a Stripe PaymentIntent is created with `payment_method_types: ['oxxo']` and OXXO billing details (name, email).
- [ ] **AC2:** The OXXO voucher (barcode image URL + reference number) is returned to the client and displayed on a confirmation screen.
- [ ] **AC3:** An email is sent to the buyer with the OXXO reference, barcode image, and instructions (see NTF-002).
- [ ] **AC4:** `TicketTier.sold` is incremented (inventory held) immediately on OXXO order creation — not on payment confirmation.
- [ ] **AC5:** Vercel Cron job (`/api/cron/oxxo-expiry`) runs every 15 minutes and releases inventory for expired OXXO orders (older than 72h, not yet paid).
- [ ] **AC6:** On OXXO payment confirmation (webhook `payment_intent.succeeded`), tickets are generated and the buyer receives a confirmation email with QR tickets (see NTF-003).
- [ ] **AC7:** On OXXO expiry, the buyer receives an "expiry" email (see NTF-002).
- [ ] **AC8:** The OXXO convenience fee (MXN 13) is displayed and added to the buyer's total when OXXO is selected.

## Technical Notes

### OXXO Payment Flow

```
1. Buyer selects OXXO → tRPC order.create → hold inventory
2. tRPC payment.createIntent(method: 'oxxo'):
   - PaymentIntent with payment_method_types: ['oxxo']
   - payment_method_data: { type: 'oxxo', billing_details: { name, email } }
   - Stripe generates OXXO voucher (barcode + reference)
   - Store oxxoReference, oxxoBarcode, oxxoExpiresAt (= now + 72h)
   - Return voucher data to client
3. Client shows OXXO reference screen (Phase 3 § B-07)
4. Send OXXO reference email (NTF-002)
5. Buyer pays at OXXO store → Stripe webhook: payment_intent.succeeded
   - confirm order + issue tickets + send email (NTF-003)
6. Cron (every 15 min): check oxxoExpiresAt < now AND Payment.status = 'pending'
   - Set Order.status = 'expired'
   - Decrement TicketTier.sold (release inventory)
   - Send expiry email (NTF-002)
```

## Notes

- The 72h window starts from when the OXXO PaymentIntent is created, not when the buyer views the reference.
- Race condition: if buyer pays at OXXO store just as the cron fires, the cron will try to expire an already-confirmed order. Handle this by checking `Payment.status = 'pending'` in the cron query — confirmed payments have `status = 'completed'` and are excluded.
- OXXO is a cash payment — there are no chargebacks (cash doesn't "reverse"). This eliminates a major fraud vector.
- The OXXO barcode image is generated by Stripe and returned as a URL. Display it as an image in the reference screen and email.
- Consider showing a countdown timer on the OXXO reference screen: "Tienes 71h 32m para completar tu pago."
